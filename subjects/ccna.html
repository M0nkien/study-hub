Cisco príkazy

Packet Tracer – kompletný prehľad konfigurácie

Táto časť obsahuje najpoužívanejšie príkazy na konfiguráciu switchov a routerov v Cisco Packet Traceri. Príkazy sú rozdelené podľa toho, čo chceš nastaviť.

1. Základný vstup do konfigurácie

enable
configure terminal
hostname S1

end
copy running-config startup-config

enable prepne do privilegovaného režimu. configure terminal otvorí globálnu konfiguráciu. copy running-config startup-config uloží konfiguráciu.

2. Heslá, šifrovanie a banner

enable
configure terminal

enable secret class

line console 0
password cisco
login
exit

line vty 0 15
password cisco
login
exit

service password-encryption
banner motd #Unauthorized access is prohibited#

end
copy running-config startup-config

Toto je základné zabezpečenie zariadenia. Používa sa na switchi aj routeri.

3. Nastavenie IP adresy na switchi – VLAN 1 / SVI

enable
configure terminal

interface vlan 1
ip address 192.168.1.2 255.255.255.0
no shutdown
exit

ip default-gateway 192.168.1.1

end
copy running-config startup-config

Switch potrebuje IP adresu na manažment. Default gateway sa nastaví, aby sa switch dal spravovať z inej siete.

4. Nastavenie IP adresy na router rozhraní

enable
configure terminal

interface gigabitEthernet 0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

interface gigabitEthernet 0/1
ip address 192.168.2.1 255.255.255.0
no shutdown
exit

end
copy running-config startup-config

Na routeri sa IP adresa nastavuje priamo na fyzické rozhranie. Nezabudni na no shutdown.

5. Popis rozhrania a vypnutie rozhrania

enable
configure terminal

interface fastEthernet 0/1
description PC1 connection
no shutdown
exit

interface fastEthernet 0/24
description UNUSED PORT
shutdown
exit

end

description pomáha pri orientácii. Nepoužívané porty je dobré vypnúť cez shutdown.

6. Vytvorenie VLAN

enable
configure terminal

vlan 10
name STUDENTI
exit

vlan 20
name UCITELIA
exit

vlan 99
name MANAGEMENT
exit

end
show vlan brief

VLAN logicky rozdeľuje sieť. Príkaz show vlan brief overí vytvorené VLAN.

7. Priradenie portu do VLAN – access port

enable
configure terminal

interface fastEthernet 0/1
switchport mode access
switchport access vlan 10
exit

interface fastEthernet 0/2
switchport mode access
switchport access vlan 20
exit

end
show vlan brief

Access port patrí do jednej VLAN a typicky je k nemu pripojený počítač.

8. Nastavenie trunk portu medzi switchmi

enable
configure terminal

interface gigabitEthernet 0/1
switchport mode trunk
switchport trunk native vlan 99
switchport trunk allowed vlan 10,20,99
exit

end
show interfaces trunk

Trunk prenáša viac VLAN naraz. Používa sa medzi switchmi alebo medzi switchom a routerom.

9. Router-on-a-stick – inter-VLAN routing

enable
configure terminal

interface gigabitEthernet 0/0
no shutdown
exit

interface gigabitEthernet 0/0.10
encapsulation dot1Q 10
ip address 192.168.10.1 255.255.255.0
exit

interface gigabitEthernet 0/0.20
encapsulation dot1Q 20
ip address 192.168.20.1 255.255.255.0
exit

interface gigabitEthernet 0/0.99
encapsulation dot1Q 99 native
ip address 192.168.99.1 255.255.255.0
exit

end
copy running-config startup-config

Router-on-a-stick umožní komunikáciu medzi VLAN cez subrozhrania routera.

10. DHCP server na routeri

enable
configure terminal

ip dhcp excluded-address 192.168.10.1 192.168.10.20

ip dhcp pool VLAN10
network 192.168.10.0 255.255.255.0
default-router 192.168.10.1
dns-server 8.8.8.8
exit

ip dhcp pool VLAN20
network 192.168.20.0 255.255.255.0
default-router 192.168.20.1
dns-server 8.8.8.8
exit

end
show ip dhcp binding
show ip dhcp pool

DHCP prideľuje IP adresy automaticky. Excluded-address vyhradí adresy, ktoré sa nemajú prideľovať klientom.

11. Statická routa

enable
configure terminal

ip route 192.168.2.0 255.255.255.0 10.0.0.2

end
show ip route

Statická routa hovorí routeru, kadiaľ posielať pakety do cudzej siete.

12. Default route

enable
configure terminal

ip route 0.0.0.0 0.0.0.0 10.0.0.2

end
show ip route

Default route sa používa, keď router nepozná konkrétnu cestu do cieľovej siete.

13. RIP v2 – jednoduchý dynamický routing

enable
configure terminal

router rip
version 2
no auto-summary
network 192.168.1.0
network 192.168.2.0
network 10.0.0.0
exit

end
show ip protocols
show ip route

RIP je jednoduchý dynamický smerovací protokol. V CCNA sa používa na precvičenie princípu routingu.

14. OSPF základná konfigurácia

enable
configure terminal

router ospf 1
router-id 1.1.1.1
network 192.168.1.0 0.0.0.255 area 0
network 10.0.0.0 0.0.0.3 area 0
exit

end
show ip ospf neighbor
show ip route

OSPF používa wildcard masku. Area 0 je chrbticová oblasť.

15. SSH prístup na switch alebo router

enable
configure terminal

hostname S1
ip domain-name studyhub.local
username admin secret admin123

crypto key generate rsa
1024

line vty 0 15
transport input ssh
login local
exit

ip ssh version 2

end
show ip ssh

SSH je bezpečnejší vzdialený prístup ako Telnet. Vyžaduje hostname, domain-name, používateľa a RSA kľúč.

16. Port security na switchi

enable
configure terminal

interface fastEthernet 0/1
switchport mode access
switchport port-security
switchport port-security maximum 1
switchport port-security mac-address sticky
switchport port-security violation shutdown
exit

end
show port-security
show port-security interface fastEthernet 0/1

Port security obmedzí, ktoré MAC adresy sa môžu pripojiť na port.

17. EtherChannel základ

enable
configure terminal

interface range gigabitEthernet 0/1 - 2
channel-group 1 mode active
exit

interface port-channel 1
switchport mode trunk
exit

end
show etherchannel summary

EtherChannel spojí viac fyzických liniek do jednej logickej linky. Mode active používa LACP.

18. Kontrolné show príkazy

show running-config
show startup-config
show ip interface brief
show interfaces
show interfaces trunk
show vlan brief
show mac address-table
show ip route
show cdp neighbors
show cdp neighbors detail
show version
show clock

Tieto príkazy používaj na kontrolu konfigurácie, hľadanie chýb a overenie konektivity.

19. Ping a traceroute

ping 192.168.1.1
ping 8.8.8.8
traceroute 192.168.2.10

Ping overí dostupnosť zariadenia. Traceroute ukáže cestu paketov k cieľu.

20. Vymazanie konfigurácie a reload

enable

erase startup-config
reload

delete vlan.dat
reload

Používa sa pri resetovaní zariadenia v Packet Traceri. Na switchi treba často zmazať aj vlan.dat.
